## Regeneration

These sample files are rederivable from the corresponding `grammar_<id>.json`
files plus the experiment sampling configuration in `main.py`.

For `exp_agreement`, the relevant sampling parameters are:

- experiment name: `agreement`
- grammar sizes: `[25, 50, 100, 1000, 5000, 7500, 10000]`
- agreement configs:
  - `agreement_enabled_a=False, agreement_enabled_b=False`
  - `agreement_enabled_a=False, agreement_enabled_b=True`
  - `agreement_enabled_a=True, agreement_enabled_b=False`
  - `agreement_enabled_a=True, agreement_enabled_b=True`
- grammars per size/config: `2`
- sample RNG seed: the grammar seed returned by `deterministic_seed(...)`
- depth range: `0..5`
- samples per depth: `20`

The experiment entrypoint is:

```bash
uv run python main.py exp_agreement
```

For a single grammar, samples are generated by:

```bash
uv run python main.py gen_samples --grammar_name=<grammar_id> --rng_seed=<seed> --min_depth=0 --max_depth=5 --n_samples_per_depth=20
```

The sample rows themselves also record:

- `grammar_name`
- `rng_seed`
- `min_depth`
- `max_depth`

So the large omitted files should be reproducible as long as the grammar JSON,
sampling seed, and sampler implementation remain unchanged.
